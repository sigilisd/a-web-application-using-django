# Новости на Django

Небольшое веб-приложение для публикации новостей на Django. Хранение — SQLite, управление зависимостями — uv. В проекте есть админка для управления статьями и публичная лента с поиском по заголовку.

## Что реализовано
- Модель `Article` с полями: `title`, `content`, `author`, `created_at`, `updated_at`, `is_published`.
- Админ-панель Django для управления новостями.
- Публичная главная страница со списком последних опубликованных новостей и фильтром по заголовку.
- Страница новости с полным текстом и данными об авторе/дате.
- Представления на CBV, базовый CSS встроен в `templates/base.html`.

## Требования
- Python 3.10+
- uv (менеджер пакетов и виртуальной среды)

Если uv не установлен:
```bash
python3 -m pip install --user uv
```
Убедитесь, что `~/.local/bin` есть в `PATH`.

## Настройка проекта
1. Клонируйте репозиторий и перейдите в каталог проекта.
2. Установите зависимости и создайте виртуальную среду:
   ```bash
   uv sync
   ```
3. Создайте файл окружения:
   ```bash
   cp .env.example .env
   ```
   Заполните значения при необходимости:
   - `SECRET_KEY` — секретный ключ Django.
   - `DEBUG` — `True` или `False`.
   - `ALLOWED_HOSTS` — хосты через запятую (например `localhost,127.0.0.1`).
4. Примените миграции и создайте суперпользователя (по желанию):
   ```bash
   uv run python manage.py migrate
   uv run python manage.py createsuperuser
   ```
5. Запустите сервер разработки:
   ```bash
   uv run python manage.py runserver
   ```
   Главная страница доступна на `http://127.0.0.1:8000/`, админка — на `/admin/`.

## Структура
- `config/` — настройки проекта и маршруты.
- `news/` — приложение с моделями, представлениями, шаблонами и админкой.
- `templates/` — общий базовый шаблон.

## Проверка
- Проверка: `uv run python manage.py check`.
- База данных по умолчанию — `db.sqlite3` в корне проекта.
